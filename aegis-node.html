Aegis Node – ESP32 Wearable Safety System

March 2026  •  ESP32, MAX30102, MPU-6050, MQ-2, SSE, Chart.js

ESP32-based wearable safety device integrating MAX30102 (heart rate/SpO2), MPU-6050 (fall detection), and MQ-2 (gas sensor) with an OLED display and buzzer alerts. Features a real-time SSE-based web dashboard with Chart.js waveforms and tiered safety alert logic for continuous health and environment monitoring.

Problem Statement

Why this project?

Workers in hazardous environments — construction sites, factories, mines, and warehouses — face risks from sudden falls, undetected gas leaks, and unnoticed health emergencies such as cardiac stress. Most safety gear is passive and provides no continuous, real-time monitoring or remote visibility for supervisors.

There is a need for an affordable, wearable device that can continuously track vital signs, detect falls, monitor surrounding air quality, alert the wearer instantly, and stream this information to a remote dashboard so that incidents can be responded to immediately.

The Project

What I built

An ESP32-based wearable safety system for real-time health and environment monitoring.

Aegis Node integrates a MAX30102 pulse oximeter, an MPU-6050 IMU for fall detection, and an MQ-2 gas sensor, with live data shown on an onboard OLED and streamed to a browser dashboard via Server-Sent Events (SSE) with animated Chart.js waveforms and tiered safety alerts.

Key Features

  • MAX30102 for real-time heart rate and SpO2 monitoring
  • MPU-6050 IMU with fall detection algorithm
  • MQ-2 gas/smoke sensor with threshold-based alerts
  • OLED display for local real-time readouts
  • SSE-based real-time web dashboard with Chart.js waveforms
  • Tiered buzzer safety alert system

Technologies Used

ESP32 MAX30102 MPU-6050 MQ-2 SSE Chart.js Embedded C Arduino IDE

What's Next

Future Advancements

  • GPS integration — every worker's real-time location tracked on a live map, so incident locations are known instantly
  • RFID-based multi-user system — each worker carries an RFID tag, letting the central dashboard identify who, where, and what's wrong across an entire workforce simultaneously
  • Automated emergency dispatch — if vitals cross critical thresholds, the system autonomously triggers an alert to the nearest hospital and ambulance service with the worker's live GPS location attached
